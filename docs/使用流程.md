# EasyPaper Killer 使用流程

## 1. 设计目标

本流程的设计重点是让用户主要通过和 agent 对话来完成论文写作任务。用户只需要在必要时准备资料并放入指定目录，其余流程由 agent 主动推进。

这里的 agent 默认是指 Claude Code、Codex、OpenCode 这类已经进入本目录工作的工具，而不是项目内另起一个必须依赖外部 API 的独立服务。

## 2. 标准交互流程

### 阶段一：任务建立

用户在对话中提供：

- 课程名称
- 大致选题方向
- 字数要求
- 截止时间
- 是否需要摘要、关键词、致谢、英文摘要
- 是否已有模板

agent 在这一阶段需要完成：

- 判断题目是否过大、过空或过难。
- 给出 2 到 5 个更适合写作的题目方向。
- 和用户确认最终题目。
- 如信息不足，主动追问缺失约束。

agent 结束这一阶段时，必须告诉用户：

- 当前已确认的信息
- 仍缺少的信息
- 下一步要不要开始生成检索式

### 阶段二：检索准备

agent 需要输出：

- 推荐检索关键词
- 知网检索式
- 筛文建议
- 建议下载的文献数量

agent 必须明确提示用户把资料放到以下位置：

- PDF：`input/references/raw/pdf/`
- 查新引文和参考文献文本：`input/references/raw/`
- 如果有额外写作要求：`input/constraints/格式与写作要求.md`

agent 结束这一阶段时，必须告诉用户：

- 需要准备哪些资料
- 每类资料该放到哪里
- 资料放好后应该回复什么

建议提示模板：

```text
你下一步需要做的是准备文献资料。
1. 下载相关论文 PDF，放到 input/references/raw/pdf/
2. 把查新引文和参考文献文本放到 input/references/raw/
3. 如果老师有额外格式要求，补充到 input/constraints/格式与写作要求.md
完成后回复我：资料已放入。
```

### 阶段三：资料入库

用户回复“资料已放入”后，agent 需要：

- 检查资料是否齐全。
- 解析文献元数据。
- 粗略判断资料是否足以支撑当前题目。
- 输出资料盘点结果。

如果资料不足，agent 不能继续硬写，必须明确指出：

- 缺的是什么类型的文献
- 推荐再补几篇
- 建议的补充方向是什么

### 阶段四：大纲确认

资料合格后，agent 需要：

- 生成论文大纲
- 给出每章的写作目标
- 标出每章主要依赖哪些文献
- 说明哪里可能还需要补资料

用户需要在这一阶段确认：

- 大纲结构是否满意
- 是否需要调整章节顺序
- 是否需要加入特定内容

agent 在这一阶段结束时，必须提示：

- 如果大纲确认，将进入正文写作
- 如果要修改，请用户指出改动要求

### 阶段五：分章写作

agent 应尽量按章节推进，而不是直接一次性生成整篇终稿。

默认执行方式是：agent 读取 `workspace/当前项目/prompts/` 下的章节 prompt，逐章逐批完成写作，并把结果回填到 `workspace/当前项目/section_drafts/`。

每次写作时，agent 应说明：

- 当前正在写哪一章
- 本章主要使用哪些文献
- 当前是否存在证据不足风险

如果某章资料明显不足，agent 应暂停该章并要求用户补资料。

### 阶段六：终稿整理

正文完成后，agent 需要：

- 生成摘要与关键词
- 整理参考文献列表
- 检查正文和引用的一致性
- 生成校验报告
- 按模板导出 docx

## 3. agent 的固定输出要求

除纯讨论场景外，agent 的阶段性答复应尽量包含以下内容：

- 当前阶段
- 已完成事项
- 当前发现的问题
- 用户下一步动作
- 文件放置路径
- 完成后的回复口令

## 4. 用户最少操作原则

为了保证体验，项目应尽量遵守以下约束：

- 不要求用户频繁手动改中间文件。
- 不要求用户理解复杂项目结构。
- 只在必要时让用户做“放资料”和“确认决策”两类动作。
- 所有下一步操作都由 agent 明确说明。
- 不默认要求用户额外配置外部模型 API 才能使用主流程。

## 5. 异常情况处理

### 资料目录为空

agent 应提示用户准备资料，而不是继续生成论文。

### 模板缺失

agent 可以先生成内容，但必须明确说明当前尚未按正式模板排版。

### 文献与题目不匹配

agent 应优先建议缩小题目或补资料，而不是扩写空泛内容。

### 需要新增文献

agent 可以给出候选文献建议，但必须提醒用户补齐资料后才能纳入正式引用。
