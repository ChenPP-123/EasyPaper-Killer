# EasyPaper Killer 项目设计

## 1. 项目定位

本项目是一个证据驱动的论文写作工作环境，主要服务于大学课程论文场景，重点解决以下问题：

- 用户不知道如何把一个宽泛选题收敛成可写题目。
- 用户检索到资料后，难以快速整理出清晰大纲。
- 用户需要满足学校模板和参考文献格式要求。
- 用户希望主要通过和 agent 对话完成流程，而不是手动操作一堆文件。

本项目默认服务对象不是普通最终用户程序，而是 Claude Code、Codex、OpenCode 等能直接在目录中工作的 agent 工具。本项目不追求“完全自动替代人”，而是追求人机协同下的高效率和低出错率。

## 2. 核心原则

- 对话优先：用户的大部分操作体验应当发生在和 agent 的对话中。
- 文件最少：除必要资料导入外，不要求用户频繁手动编辑文件。
- 证据优先：每个关键事实和判断都应尽量能追溯到已提供资料。
- 模板优先：最终输出需要尽量贴近用户提供的论文模板。
- 显式下一步：agent 在每个阶段都要告诉用户下一步该做什么。
- agent 原生优先：默认由进入目录的 agent 直接读取和更新这些文件，而不是默认依赖额外的外部 API 编排层。

## 3. 系统目标

### 3.1 用户视角目标

用户进入项目后，理想体验应当类似：

1. 用户说出大致选题和论文要求。
2. agent 给出更适合写作的题目建议，并要求用户确认。
3. agent 生成检索式，并明确告诉用户去下载什么资料、放到哪个目录。
4. 用户把资料放进去后，只需回到对话里告诉 agent“我放好了”。
5. agent 自动继续完成解析、大纲、写作、校验和导出。

### 3.2 工程视角目标

- 输入资料结构化。
- 写作过程可分阶段讨论。
- 正文和引用关系可校验。
- 输出文档可套用模板。
- 即使没有额外模型 API，也能由当前 agent 会话直接推进主流程。

## 4. 模块划分

### 4.1 交互编排模块

负责控制对话式流程，判断当前处于哪个阶段，并向用户发出明确的下一步指令。默认这部分能力由当前正在目录中工作的 agent 承担。

输出示例：

- 当前阶段：等待资料导入
- 你下一步需要做的事：把 PDF 放到 `input/references/raw/pdf/`
- 完成后回复：`资料已放入`

### 4.2 资料导入模块

负责接收并识别以下输入：

- 选题需求
- 查新引文文本
- 参考文献文本
- PDF 原文
- 模板 docx
- 老师或课程特殊要求

### 4.3 资料解析模块

负责把原始资料转换为结构化数据，包括：

- 文献基础信息
- 摘要与核心观点
- 候选证据片段
- 参考文献格式化条目

### 4.4 大纲规划模块

负责根据题目和资料生成章节结构，确保每章有明确的写作目标和可用文献范围。

### 4.5 章节写作模块

负责按章节分批生成正文内容。模型承担主要写作工作，规则层只负责限制可用证据、当前章节范围和单次生成长度，避免整篇一次性生成导致跑偏或触发输出 token 上限。默认由当前 agent 读取 `workspace/当前项目/prompts/` 下的 prompt 包逐批执行。

### 4.6 引用管理模块

负责：

- 为正文内容绑定引用编号
- 维护正文与参考文献列表的一致性
- 对缺失引用或孤立文献发出提醒

### 4.7 模板输出模块

负责按模板要求导出 docx，包括：

- 标题层级
- 字体字号
- 行间距
- 参考文献格式

### 4.8 质量校验模块

负责输出校验报告，至少覆盖：

- 缺证据内容
- 引用不一致
- 资料不足章节
- 模板未满足项

## 5. 用户与 agent 的职责边界

### 5.1 用户负责

- 提供论文方向、课程要求和字数限制。
- 确认最终题目和论文大纲。
- 去知网或其他数据库检索、筛选、下载资料。
- 把资料放入指定目录。
- 对终稿承担最终学术责任。

### 5.2 agent 负责

- 主动引导整个写作流程。
- 告诉用户当前阶段和下一步动作。
- 解析资料、生成大纲、撰写正文、整理引用。
- 在资料不足时明确提醒用户补充，而不是假装完成。
- 若无额外 API 配置，仍应直接利用当前 agent 会话完成分批写作，而不是将其视为阻塞条件。

## 6. MVP 范围

第一版应实现：

- 选题收敛
- 检索式生成
- 资料导入约定
- 文献结构化整理
- 大纲生成
- 分章写作
- 参考文献整理
- 基础校验报告
- 模板化导出

第一版不强求：

- 自动联网抓取全文
- 复杂图表生成
- 多模板通用自动适配
- 高级事实核验和查重优化

## 7. 交互设计要求

agent 的回答应尽量遵循以下格式：

1. 当前进度
2. 我已经完成了什么
3. 你下一步需要做什么
4. 资料应放到哪里
5. 你完成后应该回复什么

示例：

```text
当前进度：选题已基本确定。
我已经完成：整理出 3 个更聚焦的论文题目，并生成了检索式。
你下一步需要做：去知网检索并下载 5-10 篇最相关论文。
资料放置位置：
- PDF 放到 input/references/raw/pdf/
- 查新引文和参考文献文本放到 input/references/raw/
完成后回复：资料已放入。
```

## 8. 后续扩展方向

- 自动生成候选补充文献清单
- 更强的 PDF 内容提取与证据切片
- 多轮重写和局部改写能力
- 多学校模板适配器
- 输出证据追溯视图
