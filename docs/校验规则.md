# 校验规则

## 1. 目标

本文件用于定义论文生成过程中的最低质量门槛。校验规则的目的是防止出现无来源内容、错误引用、结构缺失和模板不符合等问题。

## 2. 内容校验

### 2.1 事实与论断校验

- 凡涉及具体事实、结论、政策判断、研究现状概括的内容，应尽量有明确来源支持。
- 没有来源支持的关键论断，不应直接进入终稿。
- 如果内容只能作为常识性过渡语句，可以不强制逐句标注，但不能承载核心信息增量。

### 2.2 章节相关性校验

- 每章内容应服务于该章既定写作目标。
- 如果某一段与章节目标明显无关，应提示调整或删除。
- 如果某章为了凑字数出现明显空泛扩写，应标记为风险项。

### 2.3 资料充分性校验

- 如果当前文献无法支撑既定题目，应优先提示用户补资料或缩小题目。
- agent 不得在资料明显不足时假装完成高质量正文。

## 3. 引用校验

### 3.1 正文到参考文献的映射

- 正文中出现的引用编号必须在参考文献列表中存在。
- 引用编号顺序应尽量保持一致和可解释。

### 3.2 参考文献到正文的映射

- 参考文献列表中的正式文献应尽量在正文中被使用。
- 如果存在未被正文使用的条目，应提示用户决定是否删除或补引。

### 3.3 候选文献限制

- 联网搜索得到但尚未补齐资料的文献只能记为候选。
- 候选文献不得直接进入正式参考文献列表。
- 候选文献不得直接作为正文正式引用依据。

## 4. 结构校验

应根据模板和任务要求，检查是否具备以下部分：

- 题目
- 摘要
- 关键词
- 正文
- 结论
- 参考文献
- 致谢或英文摘要等可选部分

如果某部分缺失，应在校验报告中明确指出。

## 5. 模板校验

至少检查以下项目：

- 标题层级是否完整
- 正文字体字号是否符合要求
- 行距和段距是否符合要求
- 参考文献格式是否符合要求
- 是否缺少模板要求的固定章节

如果模板尚未解析完成，应在导出说明中显式声明。

## 6. 输出等级

### 6.1 可继续写作

满足以下条件时可继续推进：

- 资料基本齐全
- 大纲结构合理
- 大部分章节可找到支撑文献

### 6.2 可生成初稿

满足以下条件时可输出初稿：

- 大纲已确认
- 正文主体完整
- 引用关系基本成立
- 已知问题已写入校验报告

### 6.3 可导出终稿

满足以下条件时才适合导出终稿：

- 关键论断均有支撑
- 引用和参考文献一致
- 结构齐全
- 模板要求已尽量满足

## 7. agent 行为约束

agent 在触发以下情况时，必须主动提醒用户，而不是继续沉默推进：

- 当前资料不足以支撑选题
- 某章节证据明显不足
- 模板缺失或格式要求不明确
- 需要新增文献才能完成论证

提醒时应包含：

- 问题是什么
- 为什么会影响写作质量
- 用户下一步该做什么
- 资料应放在哪里

## 8. 校验报告建议格式

建议至少包含以下栏目：

- 已完成检查项
- 风险项
- 必须修复项
- 可选优化项
- 用户下一步动作
