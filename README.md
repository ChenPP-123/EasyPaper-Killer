# EasyPaper Killer

> 把论文从"地狱模式"变成聊天模式。

## 一句话说清楚

打开终端，告诉 AI 你要写什么论文，然后跟着它一步步走——选题、大纲、正文、引用、排版，它全包了。你要做的只有三件事：**说话、确认、把资料拖进文件夹**。

```
你: 我要写一篇关于碳中和的课程论文，5000 字
AI: 好，我帮你收敛了三个选题方向，你看哪个合适？
你: 第二个
AI: 题目确定了。现在去知网搜这几个关键词，下载 5-10 篇 PDF，放到 input/references/raw/pdf/
你: （拖文件）放好了
AI: 资料已盘点，覆盖够。大纲出来了，你看章节结构可以吗？
你: 第三章和第四章换一下
AI: 改好了。开始写第一章……
（几轮对话后）
AI: 全部写完了，docx 已导出在 output/论文初稿.docx
```

## 它不是什么

- **不是**论文生成器——不会凭空编造内容，所有事实必须追溯到你的资料
- **不是** ChatGPT 包装——它是给 Claude Code、Codex、OpenCode 这些 AI 编程助手用的工作台，它们进入这个目录后就能接管写作流程
- **不是**一键出稿——它是分阶段、可审核、可干预的协作过程

## 它能写什么

| 适合 | 不太适合 |
|------|----------|
| 课程论文 | 需要原创实验数据的论文 |
| 综述型论文 | 公式推导密集的文章 |
| 政策分析 / 现状对策 | 纯数学建模 |
| 中文为主，英文为辅 | 需要有外部 API 才能推进 |

## 怎么开始

### 你需要准备

1. 一个 AI 编程助手（Claude Code / Codex / OpenCode）
2. 进入这个目录
3. 说"我要写论文"

就这些。不用装 Python 包，不用配环境，不用学命令。

### AI 助手进入后会自动

- 读 `AGENTS.md` 了解行为约定
- 跑 `python -m src.app init` 判断是新任务还是继续上次
- 跑 `python -m src.app guide` 告诉你下一步该做什么
- 引导你走完 7 个阶段

## 完整流程

```
1. 选题收敛     → 你给方向，AI 给候选题目
2. 检索准备     → AI 出检索式，你去找资料
3. 资料盘点     → AI 解析文献，判断够不够
4. 大纲生成     → AI 出大纲，你确认或调整
5. 分批写作     → 按章节逐批写，每批可审可改
6. 整理校验     → 引用排版、摘要生成、校验报告
7. 导出交付     → 生成 docx，引用自动上标
```

每一步之间都有"闸门"：资料不够就不能硬写大纲，大纲没确认就不会动笔。不会假装完成。

## 目录结构

```
EasyPaper-Killer/
├── AGENTS.md           # AI 助手的行为守则
├── README.md           # 你正在看的
├── docs/               # 设计文档（好奇的时候翻）
├── input/              # 你放资料的地方
│   ├── topic/          #   论文需求和方向
│   ├── constraints/    #   额外约束
│   └── references/     #   文献 PDF 和引文
├── template/           # 你的格式模板（可选）
├── workspace/          # AI 的工作笔记
│   └── 当前项目/       #   当前进行中的论文
├── output/             # 最终导出的 docx
└── src/                # 工具脚本（不用管）
    ├── export/office/  #   docx 高级编辑：拆包/打包/校验/批注
    └── writing/nature/ #   Nature 风格写作指引
```

## 亮点

### 引用自动上标
导出的 docx 中，所有 `[1]` `[2]` `[1-3]` 格式的引用会自动变为上标。参考文献列表的编号也会同步处理。不需要手动设置。

### Nature 风格写作引擎
内置 Nature 期刊风格的写作指引系统，按章节类型、论文类型、语言方向自动加载对应的写作规则。写引言有引言的规范，写结论有结论的约束。

### 零依赖 docx 编辑工具
除了基本导出，还提供高级文档编辑能力：拆包查看 OOXML 源码、修改后重新打包、校验文档结构、接受修订标记、添加批注——全部用 Python 标准库实现，不额外安装任何东西。

### 流程闸门
每个阶段之间都有显式闸门。AI 不会在资料不够的时候硬写正文，不会在大纲没确认的时候开始动笔。每一步都让你确认后再推进。

## 写给 AI 助手

Hi agent! 进入目录后：

```bash
# 判断当前状态
python -m src.app init --project-root <用户的项目路径>

# 获取下一步指引
python -m src.app guide --project-root <用户的项目路径>

# 自动推进到下一个稳定阶段
python -m src.app advance --project-root <用户的项目路径>
```

详细的写作规则、阶段定义、禁止事项都在 `AGENTS.md` 里。写作阶段记得加载 `src/writing/nature/` 的指引碎片，文档编辑参考 `src/export/office/`。
