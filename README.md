# EasyPaper Killer

> 水课论文，一键杀青。把省下来的时间花在真正重要的事情上。

## 这是什么

一个帮你**快速完成不重要课程论文**的 AI 工作台。打开 Claude Code（或 Codex、OpenCode），告诉它论文要求，然后像聊天一样跟着走——选题、大纲、正文、引用、排版，它全包了。

**注意：这个工具不是用来写毕业论文、核心期刊投稿或任何严肃学术产出的。** 它的使用场景很明确：某门课要求交一篇论文但你不想花太多时间，你需要一个东西帮你快速搞定格式和架构，然后你自己审一遍、改一改，交上去完事。

> 你要做的只有三件事：**说话、确认、把资料拖进文件夹**。

## 它能写什么

| 适合 | 不太适合 |
|------|----------|
| 通识课/选修课论文 | 毕业设计、学位论文 |
| 综述型、现状分析、对策建议 | 需要原创实验、一手数据的研究 |
| 中文为主 | 需要外部 API 才能推进的 |

**如果你要写的是一篇需要投入大量心血、影响你学术声誉的重要论文——请亲手写。** 这个工具的价值在于帮你省掉那些不值得你花时间的论文。

## 怎么开始

### 你需要准备

1. 一个 AI 编程助手（Claude Code / Codex / OpenCode）
2. 进入这个目录
3. 说"我要写论文"

就这些。不用装 Python 包，不用配环境，不用学命令。

### AI 助手会自动

- 读 `AGENTS.md` 了解行为约定
- 跑 `python -m src.app init` 判断当前状态
- **先问你有没有模版要求**（文字描述或 docx 样例都行）
- 再进入选题、检索、写作等流程

## 推荐用法：代码和数据分离

为了避免论文数据混入 Git 仓库，推荐把论文工作区放到本仓库之外的独立目录：

```bash
# 1. 克隆仓库（只做一次）
git clone <repo-url> EasyPaper-Killer

# 2. 进入仓库
cd EasyPaper-Killer

# 3. 在仓库外创建论文工作区，用 --project-root 指向它
mkdir -p ~/Papers/MyPaper
python -m src.app init --project-root ~/Papers/MyPaper
```

这样：
- **仓库**里只有代码，`git pull` 不影响你的论文数据
- **论文**存在独立目录，随意换代理、升级项目都不会丢
- 所有 `input/`、`workspace/`、`output/` 都在 `~/Papers/MyPaper/` 下

如果你只是想快速尝试，不指定 `--project-root` 也可以——`.gitignore` 已经排除所有数据目录，不会污染仓库。

## 你有模版吗？

老师发的格式要求、学长学姐传的 docx 样例、自己写的样式说明——**不管什么形式**，在选题之前告诉 agent，它会从头到尾照着做。这样你就不用写完再调格式。

放在 `template/` 目录下，或者在对话里直接描述，比如：

> "标题用黑体二号加粗居中，正文宋体小四 1.5 倍行距，参考文献用五号字"

## 大致流程

```
1. 模版确认     → 有格式要求先给，没有就跳过
2. 选题收敛     → 你给方向，AI 给候选题目
3. 检索准备     → AI 出检索式，你去知网下资料
4. 资料盘点     → AI 解析文献，判断够不够
5. 大纲生成     → AI 出大纲，你确认或调整
6. 分批写作     → 按章节逐批写，每批可审可改
7. 整理校验     → 引用排版、摘要生成、校验报告
8. 导出交付     → 生成 docx，引用自动上标
```

每一步都有"闸门"：资料不够不硬写，大纲没确认不动笔。不会假装完成。

## 目录结构

```
EasyPaper-Killer/
├── AGENTS.md           # AI 助手的行为守则
├── README.md           # 你正在看的
├── docs/               # 设计文档
├── input/              # 📥 你放资料的地方
│   ├── topic/          #   论文需求和方向
│   ├── constraints/    #   额外约束（字数、截止日期等）
│   └── references/     #   文献 PDF 和引文
├── template/           # 📐 你的格式模版（放 docx 样例）
├── workspace/          # 📝 AI 的工作笔记
│   └── 当前项目/       #   当前进行中的论文
├── output/             # 📤 最终导出的 docx
└── src/                # ⚙️ 工具脚本（不用管）
    ├── export/office/  #   docx 高级编辑
    └── writing/nature/ #   Nature 风格写作指引
```

## 亮点

### 引用自动上标
导出的 docx 中，所有 `[1]` `[2]` `[1-3]` 格式的引用自动变为上标。参考文献列表的编号也会同步处理。

### Nature 风格写作引擎
即使写的是水课论文，也不妨碍用 Nature 期刊的写作规范来提升质量。内置按章节类型、论文类型、语言方向自动加载的写作规则。

### 零依赖 docx 编辑
拆包查看 OOXML 源码、修改后重新打包、校验文档结构、接受修订标记、添加批注——全部用 Python 标准库实现。

### 模版优先
在动笔之前就确认格式要求。文字描述或 docx 样例都支持。

### 流程闸门
每个阶段之间都有显式闸门。AI 不会在资料不够时硬写正文，不会在大纲没确认时动笔。

## 写给 AI 助手

Hi agent! 进入目录后：

```bash
# 判断当前状态
python -m src.app init --project-root <路径>

# 获取下一步指引
python -m src.app guide --project-root <路径>

# 自动推进到下一个稳定阶段
python -m src.app advance --project-root <路径>
```

详细规则在 `AGENTS.md`。**开始任何工作前，先问用户有没有模版要求。** 写作阶段记得加载 `src/writing/nature/` 的指引碎片，文档编辑参考 `src/export/office/`。
