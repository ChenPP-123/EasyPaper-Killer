# EasyPaper Killer

> 水课论文，一键杀青。把省下来的时间花在真正重要的事情上。

## 这是什么

一个帮你**快速完成不重要课程论文**的 AI 工作台。打开 Claude Code（或 Codex、OpenCode），告诉它论文要求，然后像聊天一样跟着走——选题、大纲、正文、引用、排版，它全包了。

**注意：这个工具不是用来写毕业论文、核心期刊投稿或任何严肃学术产出的。** 它的使用场景很明确：某门课要求交一篇论文但你不想花太多时间，你需要一个东西帮你快速搞定格式和架构，然后你自己审一遍、改一改，交上去完事。

> 你要做的只有三件事：**说话、确认、把资料拖进文件夹**。

## 5 分钟上手

### 1. 克隆仓库

```bash
git clone https://github.com/ChenPP-123/EasyPaper-Killer.git
cd EasyPaper-Killer
```

### 2. 启用隐私检查（强烈推荐）

```bash
git config core.hooksPath .githooks
```

这会激活内置的防泄漏钩子——每次 `git commit` 前自动检查是否误提交了 API Key、手机号、身份证号、邮箱或二进制文档。

### 3. 进入项目目录，打开 AI 编程助手

用 Claude Code、Codex 或 OpenCode 打开这个目录，然后说：

> "我要写一篇论文，题目是《XXX》，大概 5000 字。"

AI 会自动读取 `AGENTS.md` 了解规则，然后问你格式要求。跟着对话走就行了。

### 4. 准备好格式要求和文献资料

AI 会在对应阶段提示你。所有资料放进 `input/` 和 `template/` 目录即可（这两个目录已被 `.gitignore` 排除，不会混入 Git 仓库）。

## 它能写什么

| 适合 | 不太适合 |
|------|----------|
| 通识课/选修课论文 | 毕业设计、学位论文 |
| 综述型、现状分析、对策建议 | 需要原创实验、一手数据的研究 |
| 中文为主 | 需要外部 API 才能推进的 |

**如果你要写的是一篇需要投入大量心血、影响你学术声誉的重要论文——请亲手写。** 这个工具的价值在于帮你省掉那些不值得你花时间的论文。

## 你有模版吗？

老师发的格式要求、学长学姐传的 docx 样例、自己写的样式说明——**不管什么形式**，在选题之前告诉 agent，它会从头到尾照着做。这样你就不用写完再调格式。

放在 `template/` 目录下，或者在对话里直接描述，比如：

> "标题用黑体二号加粗居中，正文宋体小四 1.5 倍行距，参考文献用五号字"

**这一步很关键：模版信息确认后，后续写作和导出都会遵守格式约束。**

## 大致流程

```
1. 模版确认     → 有格式要求先给，没有就跳过
2. 选题收敛     → 你给方向，AI 给候选题目
3. 检索准备     → AI 出检索式，你去知网下资料
4. 资料盘点     → AI 解析文献，判断够不够
5. 大纲生成     → AI 出大纲，你确认或调整
6. 分批写作     → 按章节逐批写，每批可审可改
7. 整理校验     → 引用排版、摘要生成、校验报告
8. 导出交付     → 生成 docx，引用自动上标
```

每一步都有"闸门"：资料不够不硬写，大纲没确认不动笔。不会假装完成。

## 目录结构

```
EasyPaper-Killer/
├── AGENTS.md           # AI 助手的行为守则
├── README.md           # 你正在看的
├── START.md            # 快速启动指引
├── docs/               # 设计文档
├── .githooks/          # 防泄漏检查钩子
├── input/              # 📥 你放资料的地方
│   ├── topic/          #   论文需求和方向
│   ├── constraints/    #   额外约束（字数、截止日期等）
│   └── references/     #   文献 PDF 和引文
├── template/           # 📐 你的格式模版（放 docx 样例）
├── workspace/          # 📝 AI 的工作笔记
│   └── 当前项目/       #   当前进行中的论文
├── output/             # 📤 最终导出的 docx
└── src/                # ⚙️ 工具脚本（不用管）
    ├── export/office/  #   docx 高级编辑
    └── writing/nature/ #   Nature 风格写作指引
```

## 亮点

### 引用自动上标
导出的 docx 中，所有 `[1]` `[2]` `[1-3]` 格式的引用自动变为上标。参考文献列表的编号也会同步处理。

### Nature 风格写作引擎
即使写的是水课论文，也不妨碍用 Nature 期刊的写作规范来提升质量。内置按章节类型、论文类型、语言方向自动加载的写作规则。

### 零依赖 docx 编辑
拆包查看 OOXML 源码、修改后重新打包、校验文档结构、接受修订标记、添加批注——全部用 Python 标准库实现。

### 模版优先
在动笔之前就确认格式要求。文字描述或 docx 样例都支持。

### 流程闸门
每个阶段之间都有显式闸门。AI 不会在资料不够时硬写正文，不会在大纲没确认时动笔。

### 隐私保护
内置 git pre-commit 和 pre-push 钩子，自动拦截 API Key、手机号、身份证号、邮箱和二进制文档的误提交。

## 常见问题

**Q: 需要安装什么吗？**
不用。不需要 pip install，不需要配 API Key，不需要装 Pandoc。打开 AI 编程助手就能用。

**Q: 我的个人信息会泄露吗？**
不会。`workspace/`、`input/`、`output/`、`template/` 下的内容已被 `.gitignore` 排除。克隆后建议执行 `git config core.hooksPath .githooks`，每次 commit 会自动检查敏感数据。

**Q: 我能直接用这个写毕业论文吗？**
不建议。这个工具的设计目标是"值得你花 2 小时的论文，而不是值得你花 200 小时的论文"。毕业论文和期刊投稿请亲手写。

**Q: AI 写的质量靠谱吗？**
AI 负责帮你搭好框架、铺好初稿，但**你需要自己审一遍、改一改再交**。把它当作一个帮你省掉 80% 机械劳动的工具，而不是一个自动驾驶的论文机器。

## 写给 AI 助手

Hi agent! 进入目录后：

```bash
# 判断当前状态（默认 project-root 为当前目录）
python -m src.app init

# 获取下一步指引
python -m src.app guide

# 自动推进到下一个稳定阶段
python -m src.app advance
```

详细规则在 `AGENTS.md`。**开始任何工作前，先问用户有没有模版要求。** 写作阶段记得加载 `src/writing/nature/` 的指引碎片，文档编辑参考 `src/export/office/`。
